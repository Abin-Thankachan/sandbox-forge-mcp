Why VM-First Isolation?

Standard Docker-on-host shares the host OS kernel, leaving your machine vulnerable to escapes and "pollution" like orphaned volumes or hanging network bridges. SandboxForge flips the script.

  • shield_lock
    No Shared Kernel Vulnerabilities Attacks targeting kernel bugs are contained within the VM, never touching your host OS.
  • delete_sweep
    Zero Host Pollution When the task finishes, the entire VM is destroyed. No leftover Docker images, caches, or volumes.
  • devices
    Full System Independence Run isolated Linux guest environments across macOS, Linux, and native Windows hosts.

Frequently Asked Questions

What OS support is currently offered? expand_more
Supported hosts:
• macOS: Lima backend (default)
• Linux: Lima backend (default)
• Windows (native): Hyper-V backend (default with SANDBOX_BACKEND=auto)
• Windows via WSL2-hosted server runtime: not supported in v1
Why a VM instead of just Docker? expand_more
Docker containers share the host machine's kernel. A security flaw in the kernel can lead to a container escape. By using a VM (via a backend VM), we provide a hard virtualization boundary: the VM has its own kernel, making it significantly harder for malicious code to reach your host system.
How is this more secure? expand_more
Security is handled in layers. First, the task is isolated in a guest VM. Second, any Docker operations happen inside that guest. This "nested" approach ensures that even if a build script tries to wipe a drive or install a rootkit, it only affects the disposable guest VM which is destroyed seconds later.
Does this require special hardware? expand_more
It requires hardware virtualization support, which is standard on modern Apple Silicon and Intel/AMD systems. Backend requirements by host:
• macOS/Linux: Lima prerequisites
• Windows: Hyper-V enabled plus OpenSSH client tools
What happens to my data? expand_more
Only the files you explicitly sync are shared with the VM. Once the VM is shut down, all temporary data inside the VM—including the OS itself—is deleted. This guarantees a "fresh start" for every single run.
